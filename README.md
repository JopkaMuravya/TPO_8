Список и описание тест-кейсов

| ID | Название | Шаги (Steps) | Ожидаемый результат (Expected Result) |
|:---|:---|:---|:---|
| 1 | Успешная авторизация | 1. Открыть сайт. 2. Ввести валидный логин. 3. Ввести валидный пароль. 4. Нажать кнопку Login. | Перенаправление на страницу каталога товаров `inventory.html`. |
| 2 | Вход заблокированным пользователем | 1. Ввести логин `locked_out_user`. 2. Ввести пароль. 3. Нажать Login. | Появление сообщения об ошибке: "Sorry, this user has been locked out". |
| 3 | Вход с неверным паролем | 1. Ввести валидный логин. 2. Ввести неверный пароль. 3. Нажать Login. | Появление сообщения об ошибке: "Username and password do not match". |
| 4 | Добавление товара в корзину | 1. Авторизоваться. 2. Нажать кнопку "Add to cart" у товара Sauce Labs Backpack. | Иконка корзины в шапке сайта обновляется, появляется счетчик "1". |
| 5 | Удаление товара через каталог | 1. Добавить товар в корзину. 2. Нажать кнопку "Remove" на карточке того же товара. | Счетчик на иконке корзины исчезает (корзина пуста). |
| 6 | Переход в карточку товара | 1. Авторизоваться. 2. Кликнуть по названию товара "Sauce Labs Backpack". | Переход на страницу с детальным описанием выбранного товара. |
| 7 | Выход из системы (Logout) | 1. Открыть боковое меню (бургер-меню). 2. Нажать на ссылку "Logout". | Завершение сессии и возврат на главную страницу авторизации. |
| 8 | Сохранение корзины при обновлении | 1. Добавить товар в корзину. 2. Обновить страницу браузера. | Счетчик товаров в корзине сохраняется и по-прежнему равен "1". |
| 9 | Проверка перехода к оформлению | 1. Авторизоваться. 2. Перейти на страницу оформления заказа. | Отображается форма ввода данных покупателя (Checkout: Your Information). |
| 10 | Валидация пустых полей | 1. Нажать кнопку Login, оставив поля логина и пароля пустыми. | Появление сообщения об ошибке: "Epic sadface: Username is required". |
